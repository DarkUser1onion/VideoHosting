# Bugfix Requirements Document

## Introduction

Исправление бага в механизме лайков/дизлайков видео-хостинг приложения. При нажатии на кнопку лайка или дизлайка количество просмотров обновляется корректно, но визуальное выделение активной кнопки не работает. Также не работает toggle-механизм (снятие лайка/дизлайка при повторном нажатии).

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN пользователь нажимает на кнопку лайка или дизлайка THEN система обновляет счётчики лайков/дизлайков на сервере, но кнопка не выделяется как активная в UI

1.2 WHEN сервер возвращает 200 OK с пустым телом (при удалении лайка при повторном нажатии) THEN клиент игнорирует этот ответ и не обновляет визуальное состояние кнопки

1.3 WHEN пользователь повторно нажимает на уже активный лайк или дизлайк THEN система должна убирать реакцию, но кнопка остаётся выделенной

1.4 WHEN сервер возвращает 404 NotFound при удалении лайка THEN клиент воспринимает это как ошибку и не обновляет статус UserLiked

### Expected Behavior (Correct)

2.1 WHEN пользователь нажимает на кнопку лайка THEN система SHALL обновлять счётчики И выделять кнопку лайка как активную

2.2 WHEN пользователь нажимает на кнопку дизлайка THEN система SHALL обновлять счётчики И выделять кнопку дизлайка как активную

2.3 WHEN сервер возвращает 200 OK с пустым телом THEN клиент SHALL интерпретировать это как успешное удаление лайка и устанавливать UserLiked = null

2.4 WHEN пользователь повторно нажимает на уже активный лайк THEN система SHALL убирать выделение с кнопки лайка и устанавливать UserLiked = null

2.5 WHEN пользователь повторно нажимает на уже активный дизлайк THEN система SHALL убирать выделение с кнопки дизлайка и устанавливать UserLiked = null

2.6 WHEN пользователь переключает лайк на дизлайк (или наоборот) THEN система SHALL снимать выделение с предыдущей кнопки и выделять новую

### Unchanged Behavior (Regression Prevention)

3.1 WHEN пользователь не аутентифицирован и нажимает на лайк/дизлайк THEN система SHALL CONTINUE TO показывать сообщение о необходимости входа в аккаунт

3.2 WHEN сервер возвращает ошибку сети или 500 THEN система SHALL CONTINUE TO показывать сообщение об ошибке и не изменять состояние UI

3.3 WHEN счётчики лайков/дизлайков обновляются THEN система SHALL CONTINUE TO корректно отображать числовые значения после обновления

3.4 WHEN видео загружается и пользователь уже ставил лайк/дизлайк THEN система SHALL CONTINUE TO загружать и отображать актуальный статус (IsLikeActive/IsDislikeActive)

3.5 WHEN сервер возвращает 404 NotFound при отсутствии видео THEN система SHALL CONTINUE TO обрабатывать это как ошибку (не связано с toggle лайка)
