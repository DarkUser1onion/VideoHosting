# История изменений проекта VideoHosting

## [2026-05-05] Исправление системы просмотра видео

### Проблемы, которые были решены

#### 1. Двойной подсчёт просмотров при открытии видео через MPV
**Проблема:** При открытии видео через MPV просмотры засчитывались дважды. Это происходило из-за того, что:
- MPV при открытии HLS потока делает несколько запросов к плейлисту
- Инкремент просмотров происходил при каждом запросе к `/stream` endpoint

**Решение:**
- Создан новый endpoint `POST /api/videos/{id}/view` для явной записи просмотра
- Убран автоматический инкремент просмотров из `/stream` endpoint
- Клиент теперь явно вызывает `/view` при реальном начале воспроизведения:
  - В браузере: JavaScript отправляет POST запрос при событии `play` (только один раз)
  - В MPV: Клиент отправляет POST запрос при открытии видео

#### 2. Ошибка базы данных при открытии видео в браузере
**Проблема:** При открытии видео в браузере сервер выдавал ошибку:
```
fail: Microsoft.EntityFrameworkCore.Database.Command[20102]
Failed executing DbCommand: UPDATE "Videos" SET "Views" = @p0 WHERE "Id" = @p1
```

**Решение:**
- Обёрнут вызов `IncrementViewsAsync` в try-catch, чтобы ошибка инкремента не блокировала открытие видео
- Добавлена обработка результата `ExecuteUpdateAsync` - если видео не найдено, ошибка не возникает

#### 3. Лишнее обновление окна при закрытии MPV
**Проблема:** При закрытии окна с видео после просмотра в MPV происходило лишнее обновление списка видео.

**Решение:**
- Добавлен флаг `_videoDeleted` в `PlayerViewModel`
- Обновление списка (`_refreshCallback`) теперь вызывается только при удалении видео, а не при каждом закрытии окна

#### 4. Повторное открытие MPV
**Проблема:** При двойном нажатии на "Открыть в MPV" могло открыться несколько окон.

**Решение:**
- Добавлена проверка `IsMpvRunning` - если MPV уже запущен, повторное нажатие игнорируется

### Изменённые файлы

#### Server (Backend)
- `Server/Controllers/VideosController.cs`:
  - Добавлен новый endpoint `POST /api/videos/{id}/view`
  - Убран автоматический инкремент из `/watch` endpoint
  - Добавлен try-catch для обработки ошибок БД
  - HTML страница `/watch` теперь отправляет POST запрос на `/view` при начале воспроизведения

- `Server/Services/VideoService.cs`:
  - Добавлена обработка результата `ExecuteUpdateAsync` в `IncrementViewsAsync`

#### Client (Frontend)
- `Model/MainWindowViewModel.cs`:
  - Добавлен метод `RecordViewAsync()` для отправки POST запроса на `/view`
  - Обновлён метод `OpenInMpv()` - теперь отправляет запрос на запись просмотра

- `Model/PlayerViewModel.cs`:
  - Добавлен флаг `_videoDeleted` для контроля обновления списка
  - Метод `OnClosed()` теперь вызывает обновление только при удалении видео

- `Views/PlayerWindow.axaml.cs`:
  - Добавлен статический метод `RecordViewAsync()`
  - Обновлён метод `OnOpenExternalRequested()` для записи просмотра

- `Views/MainWindow.axaml`:
  - Добавлена иконка колоколчика 🔔 для кнопки уведомлений
  - Добавлено превью видео и длительность в плеере

- `Views/PlayerWindow.axaml`:
  - Добавлено превью видео и длительность в плеере

### Результат
- Просмотров засчитывается **ровно один раз** за одно открытие видео
- Ошибки БД не блокируют воспроизведение видео
- Окно не обновляется лишний раз при закрытии MPV
- Повторное нажатие на "Открыть в MPV" игнорируется, если MPV уже запущен
